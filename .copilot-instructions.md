# Instruções para o GitHub Copilot - Ocean Grid Tools

## Filosofia do Projeto

Este é um **projeto pessoal** desenvolvido por um único pesquisador para auxiliar na geração de grades retangulares para modelos oceânicos. 

### Princípios Fundamentais

1. **Modularidade**: O projeto é um "pacote" (coleção de ferramentas) com funcionalidades independentes
2. **Simplicidade**: Documentação prática e objetiva, sem arquivos desnecessários
3. **Didática**: Exemplos simples para usuários com conhecimento básico de programação
4. **Pragmatismo**: Foco em funcionalidade, não em formalidades de projeto open-source

## O Que EVITAR

❌ **NÃO criar arquivos desnecessários:**
- CHANGELOG.md (é projeto pessoal, não precisa)
- CONTRIBUTING.md (desenvolvedor único)
- Documentação excessivamente formal
- Estruturas complexas de projeto open-source

❌ **NÃO usar jargão excessivo:**
- Manter linguagem técnica mas acessível
- Evitar termos que assumem conhecimento avançado

## O Que FAZER

✅ **Documentação eficiente:**
- README curtos e objetivos em cada módulo
- Exemplos práticos de uso
- Explicações do que cada script faz
- Instruções de uso passo-a-passo

✅ **Estrutura modular:**
- Cada funcionalidade em seu próprio diretório
- Dependências claras entre módulos
- Ferramentas independentes quando possível

✅ **Exemplos didáticos:**
- Código comentado em português
- Casos de uso reais
- Parâmetros bem explicados

## Convenções Específicas

### Para Grades Oceânicas

- **Formato POM**: 5 colunas (i, j, lon, lat, depth)
- **Convenção de profundidade**: depth > 0 = oceano, depth = 0 = terra
- **Espaçamento**: graus decimais (ex: 0.25° ≈ 27.8 km)

### Para Scripts Shell

- Usar `#!/usr/bin/env bash` ou `#!/bin/zsh`
- Incluir mensagens coloridas para feedback
- Verificar dependências antes de executar
- Permitir execução de qualquer diretório

## Quando Adicionar Nova Funcionalidade

1. Criar novo diretório em `tools/`
2. Seguir estrutura padrão (src/, scripts/, examples/)
3. Criar README.md sucinto no novo diretório
4. Atualizar README.md principal
5. Adicionar exemplos didáticos
